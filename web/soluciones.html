Para mitigar los problemas de seguridad se propone las siguientes soluciones:



Método de cifrado SSL este ofrece encriptación que evita que los interceptores lean los datos que transitan la nube, el cifrado SSL de 128 bits ofrece más bits de longitud de clave, además este tipo de cifrado consiste en una clave pública (que codifica la información) y una clave privada (que descifra la información), por lo que implica que sólo el poseedor de la clave pueden leerlo, esta es una adecuada opción que se puede tomar en vez de usar un cifrado SSL de 40 bits y también se puede desplazar a 256 bits siempre que sea necesario.
SSL de 128 bits es lo suficientemente complejo como para que se lleve a efecto la intromisión en los datos, conjuntamente se puede utilizar cifrado doble, uno por medio del propietario y otra mediante SSL. Por lo que se podría decir que la potencia de procesamiento que se necesita para romper la seguridad es insuficiente, logrando que la mayoría de los ataques sean ineficaces. Este enfoque no sólo preserva que los datos no sean víctima de los intrusos, sino que también ayuda a asegurar que los datos están seguros mientras están en tránsito.
Es un protocolo de estado, orientado a la conexión cliente-servidor, se podría afirmar que este protocolo de seguridad de las comunicaciones es el de mayor despliegue en Internet, que proporciona confidencialidad, integridad de datos y autenticación de la identidad para comunicaciones a través de Internet de dos partes.


Una versión actualizada y más segura de SSL es el protocolo de acceso y seguridad de datos por la red TLS (Transport Layer Security), protege los datos en tránsito entre dos puntos finales, con la autenticación de por lo menos un punto, se utiliza TLS para validar la identidad de los servidores que alojan sitios web, realizando la autenticación mutua, permitiendo a los servidores verificar la identidad de los clientes y a los clientes verificar la identidad de los servidores. Esta técnica proporciona la verificación de extremo a extremo de los resultados de una consulta a un servidor.





Los proveedores de servicio deben utilizar un método de criptografía, sin lugar a duda es un método para asegurar el transporte de datos, pero todavía tiene ciertas fallas en los mecanismos de seguridad, algunos proveedores de almacenamiento en la nube no dan información sobre el tema de seguridad a sus clientes para mantener su reputación en el mercado de la nube. El sistema de almacenamiento en la nube debe ser incorporado con SSL y servicio TCCP (Trusted Platform Cloud Computing) este es un esquema basado en un servidor seguro, está diseñado para garantizar la integridad y confidencialidad de los datos de los usuarios alojados en la nube, y determina los derechos de acceso absoluto de los usuarios.



Los datos necesitan ser comprobados después de la transmisión, para esto se hace uso de MAC (código de autenticación de mensaje) proporciona integridad de los datos ya que permite el reconocimiento de cualquier modificación o manipulación del mensaje durante la transmisión y asegura el no repudio.

Implementar OTP (contraseña de un solo uso) se generan de forma dinámica sólo con fines de uso único y tienen una vida útil limitada utilizable.


Medio de seguridad SSO (inicio de sesión único) el servicio emite una solicitud de emergencia y redirecciona al cliente IdP (proveedor de identidad), después de la autenticación correcta, el IdP emite un token de autenticación firmado y redirige al cliente de SaaS, donde se validará el token y el usuario conectado.


El CSP (proveedor de servicios de la nube) debe establecer la lista blanca de direcciones URL, es una forma eficiente de proteger la identidad del usuario debido a que si la URL no se encuentra almacenada en los registros del CSP, el acceso a la cuenta se desactivará.


Se debe exigir al proveedor que utilice el medio de contraseñas ocultas, ya que este es un archivo de contraseñas especial, el cual únicamente puede ser leído por el usuario root.


Se debería establecer la comunicación por medio de un canal seguro, estos protocolos de seguridad podrían ser: TLS, SSH o HTTPS, para asegurar que la información es transmitida de forma segura.



La copia de seguridad es sumamente importante, por la razón que no se está excento de algún desastre.




Estar al tanto de los lugares donde van a ser almacenados los datos, debido a que el ordenamiento jurídico cambia en cada país.



Los datos deben estar disponibles 24*7, sin ninguna interrupción, para ello se necesita adoptar mecanismos de equilibrio de carga efectiva durante el tratamiento de los datos.



El modelo de privilegio mínimo que se debe utilizar para los usuarios es CSP (Políticas de Seguridad de Contenido), este mecanismo determina el nivel de privilegios de un usuario en particular.



Criptografía asimétrica, proporciona una clave privada al cliente para encriptar la información, cada cliente tiene una clave privada única, y una clave pública la misma que está disponible públicamente para las personas que deseen enviar información.
